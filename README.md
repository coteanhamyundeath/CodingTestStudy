## 깃(Git) 사용 가이드

### 1️. 작업 준비 (최초 1회만)

가장 먼저 내 작업 공간(브랜치)으로 이동해야 합니다.

```bash
$ git switch 본인브랜치이름
# 예시: $ git switch user/jay-p08
```

### 2️. 코드 짜기 전 ( 항상 필수 )

코드를 작성하기 전, 다른 사람들이 올려둔 **최신 코드를 먼저 내 브랜치로** 가져와야 합니다.

```bash
git pull origin dev
```

### 3️. 코드 작성 후 올릴 때

문제를 다 풀었다면 아래 3단 콤보로 내 브랜치에 코드를 쏘아 올립니다.

```bash
# 1. 수정한 파일 모두 담기
git add .

# 2. 커밋 메시지 작성 (양식: "문제제목 / 레벨")
git commit -m "타겟 넘버 / Lv.2"

# 3. 내 브랜치로 푸시 ( dev에 바로 푸시 금지 )
git push origin 본인브랜치이름
# 예시: git push origin user/jay-p08
```

### 마무리 (PR)

푸시를 성공했다면 GitHub 웹사이트로 돌아와서 **`dev` 브랜치로 Pull Request(PR)**를 만들어주면 끝
